Join the community.

PQSafe is open-source (Apache-2.0) and openly developed. Six concrete ways to participate — pick whichever fits your time budget. All free, all public.

1 · Lowest barrier

GitHub Discussions

Open Discussions →

Ask anything: API behaviour, spec interpretation, integration questions, threat-model concerns. We respond within 48 hours. Discussions are public, indexable, and the answers help the next person. Q&A, Ideas, Show & Tell, and Polls categories are open.

2 · Real-time chat

Discord server

Join Discord →

For real-time questions, debugging, and casual chat. Founders idle in #general; #integration-help has eyes during HK working hours. Indexed transcripts are surfaced back to GitHub Discussions weekly so nothing important is lost in chat.

3 · For institutions

Post-Quantum Agent Payments Working Group

Request invite →

A monthly call + private Slack/Lark channel convening regulated FIs, payment infrastructure operators, and AI agent platform teams to coordinate on the AP2-PQ profile and cross-vendor conformance. By invitation; current participants include PQSafe + design-partner candidates from HKMA, MAS, and EU regions. Tell us your org and your role, we reply within 48 hours.

4 · Talk to a maintainer

Weekly office hours

Join the call →

Every Friday 17:00 HKT (UTC+8) for 30 minutes on the Discord voice channel. No agenda — drop in with a question or a half-built integration. Recordings posted to GitHub Discussions for asynchronous viewers in other time zones. First-come, first-served; we don't gate by org or role.

5 · Ship code

Good-first-issue tickets

Browse open tickets →

Tickets with full repro, scope, and expected diff. Pick one, comment "I'll take this", and ship. Examples currently open or planned:

  • Add Redis-backed nonce store adapter to @pqsafe/agent-pay
  • Port @pqsafe/conformance harness to Java (Bouncy Castle backend)
  • Translate /quickstart to Japanese, Korean, or Traditional Chinese
  • Add ML-DSA-87 support (NIST Level 5) alongside the default ML-DSA-65
  • Wrangler template for one-click CF Worker deploy of the verifier API
  • VS Code extension that highlights and verifies inline SpendEnvelope JSON
  • OpenAPI → Postman collection generator for the verifier API
  • Add Solana rail adapter (alongside USDC-Base) for stablecoin settlement
6 · Meet us in person

Conferences and meetups

Invite us to speak →

We accept invited talks and CFPs on AI agent payments, post-quantum cryptography, and standards work. Planned 2026 surface:

  • Money20/20 USA · CFP submitted
  • RSA Conference 2026 · CFP submitted
  • Black Hat / DEF CON 2026 · AI Village CFP
  • AI Engineer Summit · CFP open
  • ETHGlobal (HK / SG) · sponsor demo slot
  • FIDO Alliance Authenticate · standards-track session
  • Hong Kong FinTech Week 2026 · speaker

If you're organizing a meetup, podcast, or webinar where post-quantum payment authorization fits, email raymond@pqsafe.xyz. We do remote and HK / Singapore / SF in-person.

How we behave. Apache-2.0 license. No NDAs for participation. No "enterprise" tier with a sales gate — the infrastructure is free forever, paid features are operational (key custody, dashboard, multi-tenancy). We tell you when we don't know something. We tell you when PQSafe wouldn't help (see Hall of Shame). We publish what we ship and what we miss.
