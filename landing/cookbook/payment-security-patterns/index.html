Payment Security Patterns for AI Agent Frameworks

Published 2026-05-14 · PQSafe · Apache-2.0 · raw markdown

AI agents that handle payments introduce a new class of risk: compromised keys, runaway spend, hallucinated recipients, and — looking ahead — quantum adversaries capable of breaking today's ECDSA signatures. This cookbook entry shows how to wrap every agent payment action in a ML-DSA-65 SignedActionEnvelope using PQSafe, giving you cryptographically-enforced spend caps, merchant constraints, time bounds, and replay protection today.

Why This Matters: The ClawHavoc Attack

The ClawHavoc supply-chain attack of April 2026 exposed 138 CVEs across 1,400+ malicious skills on clawhub.ai. Several silently redirected payment tool calls to attacker-controlled accounts. A skill ecosystem without a mandatory verification layer is structurally unsafe. The FIDO Alliance's AP2-PQ profile (published 28 April 2026) establishes the standard pattern. This cookbook implements it.

The Threat Model

Attack Vector How SpendEnvelope Stops It
Hallucinated recipientRecipient ID is signed into envelope — mismatch at verification = block
Prompt-injected amountAmount ceiling is signed — exceeding it at verification = block
Replayed mandateNonce is single-use — server-side replay log rejects second use
Compromised package upgradeEnvelope covers all fields via canonical serialization — any tamper = invalid signature
Stale allowlistEnvelope carries explicit expiry (≤5 min recommended) — clock check = block

Recipe 1: LangChain Agent with PQSafe SpendEnvelope

import os, uuid
from datetime import datetime, timedelta, timezone
from langchain.agents import AgentExecutor, create_openai_tools_agent
from langchain.tools import Tool
from langchain_openai import ChatOpenAI
from langchain_pqsafe import PQSafeAuthorizer, SpendEnvelope

authorizer = PQSafeAuthorizer(api_key=os.environ["PQSAFE_API_KEY"])

def authorize_and_pay(charge_request: str) -> str:
    amount_cents = 2999
    merchant_id  = "stripe:acct_xxx"

    envelope = SpendEnvelope(
        agent_id="lc-agent-prod-01",
        amount_usd_cents=amount_cents,
        merchant_id=merchant_id,
        expires_at=(datetime.now(timezone.utc) + timedelta(minutes=5)).strftime("%Y-%m-%dT%H:%M:%SZ"),
        nonce=str(uuid.uuid4()),
    )

    signed = authorizer.sign(envelope)              # ML-DSA-65, 3,309-byte sig
    result = authorizer.verify(signed, endpoint="https://pqsafe.xyz/verify")
    if not result.ok:
        return f"Payment blocked by PQSafe: {result.reason}"

    # stripe.PaymentIntent.create(amount=amount_cents, currency="usd", ...)
    return f"Payment of ${amount_cents/100:.2f} authorized. Nonce: {signed.nonce}"

payment_tool = Tool(
    name="authorized_payment",
    func=authorize_and_pay,
    description="Charge a customer. Input: '<amount> to <merchant_id>'. Post-quantum authorization required.",
)

llm      = ChatOpenAI(model="gpt-4o", temperature=0)
executor = AgentExecutor(agent=create_openai_tools_agent(llm, [payment_tool], prompt=...), tools=[payment_tool])
executor.invoke({"input": "Charge the customer $29.99 for the Pro plan."})
import { AgentExecutor, createOpenAIToolsAgent } from "langchain/agents";
import { DynamicTool } from "@langchain/core/tools";
import { ChatOpenAI } from "@langchain/openai";
import { PQSafeAuthorizer, SpendEnvelope } from "@pqsafe/openclaw";
import { randomUUID } from "crypto";

const authorizer = new PQSafeAuthorizer({ apiKey: process.env.PQSAFE_API_KEY! });

const paymentTool = new DynamicTool({
  name: "authorized_payment",
  description: "Charge a customer with post-quantum authorization.",
  func: async (chargeRequest) => {
    const envelope = new SpendEnvelope({
      agentId: "lc-agent-prod-01",
      amountUsdCents: 2999,
      merchantId: "stripe:acct_xxx",
      expiresAt: new Date(Date.now() + 5 * 60_000).toISOString(),
      nonce: randomUUID(),
    });
    const signed = await authorizer.sign(envelope);
    const result = await authorizer.verify(signed, "https://pqsafe.xyz/verify");
    if (!result.ok) return `Payment blocked: ${result.reason}`;
    return `Payment of $29.99 authorized. Nonce: ${signed.nonce}`;
  },
});

Recipe 2: CrewAI Multi-Agent with Shared SpendEnvelope

A single envelope shared across agents — the nonce ensures it can only be consumed once.

from crewai import Agent, Crew, Task
from pqsafe import PQSafeAuthorizer, SpendEnvelope

authorizer = PQSafeAuthorizer(api_key=os.environ["PQSAFE_API_KEY"])

def create_shared_envelope(amount_cents: int, merchant_id: str) -> dict:
    envelope = SpendEnvelope(
        agent_id="crewai-billing-crew-v1",
        amount_usd_cents=amount_cents,
        merchant_id=merchant_id,
        expires_at=(datetime.now(timezone.utc) + timedelta(minutes=10)).strftime("%Y-%m-%dT%H:%M:%SZ"),
        nonce=str(uuid.uuid4()),
    )
    return authorizer.sign(envelope).to_dict()

shared_envelope = create_shared_envelope(amount_cents=4999, merchant_id="stripe:acct_live")

billing_agent = Agent(
    role="Billing Specialist",
    goal="Execute authorized payments only — never move money without a verified envelope.",
    allow_delegation=False,
)
billing_task = Task(
    description=f"Verify and execute this envelope: {shared_envelope}",
    agent=billing_agent,
    expected_output="Payment executed confirmation with nonce.",
)
Crew(agents=[billing_agent], tasks=[billing_task]).kickoff()

Recipe 3: OpenClaw Skill with pqsafe-pay

Install via clawhub install pqsafe-pay or npm install @pqsafe/openclaw

import { OpenClawAgent } from "@openclaw/sdk";
import { pqsafePay, SpendEnvelope } from "@pqsafe/openclaw";
import { randomUUID } from "crypto";

const agent = new OpenClawAgent({
  skills: [pqsafePay({ apiKey: process.env.PQSAFE_API_KEY!, verifyEndpoint: "https://pqsafe.xyz/verify" })],
});

agent.on("authorizePayment", async (ctx) => {
  const envelope = new SpendEnvelope({
    agentId: ctx.agentId,
    amountUsdCents: ctx.params.amountCents,
    merchantId: ctx.params.merchantId,
    expiresAt: new Date(Date.now() + 5 * 60_000).toISOString(),
    nonce: randomUUID(),
  });
  const signed = await ctx.skills.pqsafePay.sign(envelope);
  const result = await ctx.skills.pqsafePay.verify(signed);
  if (!result.ok) { ctx.reject(`PQSafe blocked: ${result.reason}`); return; }
  ctx.resolve({ authorized: true, nonce: signed.nonce });
});

What Happens When Verification Fails

Reason Code What Triggered It
SIGNATURE_INVALIDEnvelope fields tampered after signing, or wrong key used
ENVELOPE_EXPIREDexpires_at passed — prevents delayed replay attacks
NONCE_REPLAYEDNonce already consumed — prevents double-spend across threads
MERCHANT_MISMATCHmerchant_id in request doesn't match the signed envelope
AMOUNT_EXCEEDEDRequested charge exceeds signed amount_usd_cents ceiling

Security Model

Algorithm: ML-DSA-65 (NIST FIPS 204, CRYSTALS-Dilithium3 level 3)
Signature size: 3,309 bytes — covers every SpendEnvelope field
Verifier: Cloudflare Worker at pqsafe.xyz/verify — stateless, global
Replay: Server-side nonce log, TTL = expires_at + 10 minutes
RFC draft: draft-chau-ap2-pq-spend-envelope-00
Standards: FIDO Alliance Agentic Authentication TWG (AP2-PQ, Apr 28 2026)
