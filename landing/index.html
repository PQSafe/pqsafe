What we ship

Open-source SDK + Verifier API. Three building blocks.

🛤

Multi-rail agent SDK

One signed SpendEnvelope. Five payment rails: Airwallex, Wise, Stripe, USDC-Base, and x402. The agent picks the fastest or cheapest route — the authorization policy travels with it automatically.

Airwallex Wise Stripe USDC-Base x402
🔐

Cryptographic spend caps

Every envelope is ML-DSA-65 signed per NIST FIPS 204. Amount, recipient, expiry, and purpose are locked at authorization time — no envelope, no transaction. Canonical serialization per RFC 8785 JCS prevents tamper across all rails.

FIPS 204 ML-DSA-65 RFC 8785 JCS
📋

Regulator-grade audit trail

Every payment produces a tamper-evident signed envelope receipt anchored on-chain. SpendEnvelope hash, timestamp, rail, and outcome are immutable from execution. Built for HKMA QPI-aligned compliance records.
